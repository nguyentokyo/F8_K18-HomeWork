Yêu cầu bài tập

Mục tiêu: Viết toàn bộ logic JavaScript vào thẻ <script> cuối file. Không được thay đổi HTML/CSS.

1. Định nghĩa Schema với Zod TODO 1

Tạo registerSchema dùng z.object({}) với các rule sau:

  • firstName — bắt buộc, tối thiểu 2 ký tự
  • lastName — bắt buộc, tối thiểu 2 ký tự
  • email — bắt buộc, đúng định dạng email
  • phone — tuỳ chọn, nếu có phải đúng định dạng 0XXXXXXXXX (10 chữ số)
  • role — bắt buộc, phải là một trong: dev | design | pm | other
  • password — bắt buộc, ≥ 8 ký tự, có ít nhất 1 chữ cái và 1 chữ số
  • confirmPassword — bắt buộc
  • terms — phải là true

Dùng .superRefine() hoặc .refine() để kiểm tra confirmPassword === password.

2. Hàm hiển thị lỗi / thành công TODO 2

Viết hàm showError(fieldId, errorMsgId, message) và showSuccess(fieldId):

  • Thêm / xoá class is-error và is-success trên .input-field
  • Thêm / xoá class show trên .error-msg và set textContent

3. Xử lý sự kiện submit TODO 3

  • Lấy giá trị từ tất cả các field
  • Parse bằng registerSchema.safeParse(data)
  • Nếu lỗi: map error.flatten().fieldErrors → gọi showError() cho từng field
  • Nếu thành công: ẩn form, hiện #successScreen (thêm class show)

4. Validation real-time khi blur TODO 4

Với mỗi input / select, lắng nghe sự kiện blur và validate field đó ngay lập tức.

5. Toggle hiển thị mật khẩu TODO 5

Nút #togglePassword và #toggleConfirm đổi type giữa password ↔ text và đổi icon 👁 / 🙈.

6. Thanh độ mạnh mật khẩu TODO 6

Khi người dùng gõ vào #password, tính điểm (1–4) theo tiêu chí: độ dài ≥ 8, có chữ hoa, có số, có ký tự đặc biệt. Set data-level="N" trên #strengthBars và cập nhật #strengthLabel.

7. Nút Reset TODO 7

Khi click #resetBtn: reset form, xoá tất cả class lỗi/thành công, ẩn #successScreen, hiện lại form.
